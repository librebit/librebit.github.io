Instalación de servidor LAMP

Jun 12, 2014 • Iván Rodríguez

Resumen

Dentro del ámbito de servidores es común la instalación de un servidor LAMP (Linux Apache MySQL PHP). Esto debido a que existen múltiples aplicaciones que requieren la combinación de estos servicios, ya sea Moodle, MediaWiki, por mencionar algunos. Estas aplicaciones piden como prerequisitos tener instalados estos servicios para el correcto funcionamiento de ellos mismos.

A continuación se muestra la configuración de un servidor LAMP tanto para CentOS como para Ubuntu.

Selección de Linux

La selección de la distribución o sabor de Linux es importante, una página que puede ser útil en la toma de decisión es DistroWatch, en ella podemos buscar cualquier distribución de Linux disponible en línea, también se muestran las distribuciones organizadas de acuerdo a diferentes necesidades, desde un Linux para principiantes hasta un Linux especializado en análisis de seguridad o para diseño gráfico.

En lo referente a LAMP, podemos considerar que existen dos vertientes principales, los basados en distribuciones Debian/Ubuntu y los basados en RedHat/CentOS. Ubuntu se ha manejado como una distribución para usuarios que estan apenas conociendo o viviendo su experiencia en Linux, es un sistema muy versatil y trata de ser lo mas amigable al usuario, en cambio, CentOS es una distribución para gente con mas experiencia en Linux, donde la instalación de estos servicios requieren de mas pericia de parte del usuario final.

En conclusión, escoger entre Ubuntu o CentOS ya es una decisión personal, donde cada usuario determinará que tipo de sistema operativo cubre mejor sus necesidades.

Pasos para preparar un servidor CentOS

Las aplicaciones en la distribución de CentOS se instalan a través de repositorios, así como la mayoría de las distribuciones de Linux. Una forma sencilla de instalar estas aplicaciones es trabajar desde la terminal con el comando yum en modo de superusuario. Adicionalmente, los comandos de service y chkconfig se utilizan para iniciar el servicio y para que arranque automáticamente al encender el servidor.

Para iniciar en modo de superusuario, desde una terminal de CentOS se utiliza el siguiente comando:

$ su -

El signo de # en la terminal indica que estamos en modo de superusuario, cabe mencionar que en CentOS no es lo mismo su, sudo, sudo su que su -, el modo correcto es su -.

Instalación del servidor HTTP Apache

La aplicación de Apache se encuentra en los repositorios de CentOS, el comando para instalar la aplicación es:

# yum install httpd -y

Por medio de service y chkconfig se arranca el servicio y se indica que el servicio inicie automáticamente al arrancar el sistema operativo.

# service httpd start
# chkconfig httpd on

Por default, CentOS tiene deshabilitado el acceso remoto al puerto 80, para habilitar que cualquier usuario pueda ver el puerto 80 es necesario modificar el firewall del equipo. Las modificaciones en el Firewall puede realizarse sobre el archivo de iptables o de manera gráfica en System -> Administration -> Firewall y dentro de la sección de trusted services se selecciona a WWW(HTTP) 80/tcp para habilitar el servicio y reiniciamos el Firewall.

Instalación del servidor de base de datos MySQL.

Los comandos para instalar y arrancar el servicio de MySQL desde una terminal de superusuario son:

# yum install mysql mysql-server -y
# service mysqld start
# chkconfig mysqld on

Por default, CentOS no pone contraseña de root en el servidor de MySQL, para agregar una contraseña, se escribe en la terminal el siguiente comando:

Instalación de PHP

La instalación de PHP se realiza desde la terminal con el comando:

# yum install php -y

Adicionalmente se instalan los siguientes paquetes para el correcto funcionamento de PHP con los otros servicios instalados en el equipo.

# yum install php-mcrypt php-pdo php-cli php-mysql php-gd php-xml libtool-ltdl mhash mcrypt -y

De esta manera se ha instalado correctamente un servidor de HTTP, MySQL y PHP en una distribución Linux Centos.

Pasos para preparar un servidor Ubuntu

La instalación de los servicios LAMP en Ubuntu se realizan de forma similar que en CentOS, desde una ventana de terminal. La instalación de las aplicaciones desde los repositorios se realiza con el comando apt-get. El comando sudo antes de apt-get es para indicar que el comando apt-getse ejecutará con los permisos de superusuario.

Instalación del servidor HTTP Apache

Desde una terminal se escribe el comando:

$ sudo apt-get install apache2

Por default, el servidor de apache2 arranca en automatico y los puertos son liberados en el firewall, por lo que no se requiere de una configuración adicional.

Instalación del servidor de base de datos MySQL.

Para la instalación de MySQL, desde una terminal se invoca el comando:

$ sudo apt-get install mysql-server mysql-client libmysqlclient-dev

Durante el proceso de la instalación se solicita la contraseña de root para MySQL.

Instalación de PHP

Desde una terminal se escribe el comando:

$ sudo apt-get install php5 libapache2-mod-php5 php5-cli php5-mysql

Para que las modificaciones de PHP se actualicen en el servidor HTTP es necesario reiniciar Apache.

$ sudo service apache2 restart

De esta manera, se tiene instalado un servidor LAMP basado en Ubuntu.

Conclusión

Se puede observar claramente que la instalación de un servidor LAMP es mucho mas sencillo en Ubuntu, pero los niveles de seguridad y protección en datos de CentOS es algo que no podemos ignorar. Aún así, la experiencia de utilizar diferentes distribuciones de Linux para ofrecer estos servicios permite que el administrador de redes sea capaz de adaptarse a las necesidades de diferentes empresas.
